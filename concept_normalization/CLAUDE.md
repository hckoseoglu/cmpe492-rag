# Concept Normalization Module for Exercise Science RAG Pipeline

## What This Is

A pre-retrieval query normalization module that sits BEFORE the RAG vector search. It transforms ambiguous, abbreviated, or compound user queries into clean, knowledge-base-aligned forms to improve retrieval quality.

This module does NOT answer questions. It improves retrieval.

## Architecture Overview

Three submodules execute in this fixed order:

1. **Abbreviation Expansion** → expands domain abbreviations (RT → resistance training)
2. **Coordination Resolution** → decomposes compound queries ("X and Y for Z" → two queries)
3. **Term Variation Handling** → maps lay/variant terms to canonical KB forms

Each submodule receives the vocabulary JSON (extracted once from 9 knowledge-base PDFs) injected into its system prompt.

See `docs/architecture.md` for the full pipeline description, execution order justification, and data flow.

## Critical Design Constraints

- **Conservative normalization**: When in doubt, do NOT normalize. Preserve the original.
- **Knowledge-base grounded**: Every normalization must trace to a vocabulary entry. No hallucinated mappings.
- **Exercise science domain**: Do not apply general fitness normalizations.
  - "toning" → "muscular endurance training" (NOT "exercise")
  - "strength and conditioning" is ONE compound concept (do NOT decompose)
  - VO2max, 1RM, RPE are standard scientific notation — check vocabulary before expanding
- **No silent failures**: Every decision (including decisions NOT to normalize) goes in the transformation log.
- **Latency-aware**: This adds processing time before retrieval. Minimize LLM calls.

## Implementation Order

1. Run `src/vocabulary_extraction.py` against the 9 PDFs to produce `vocabulary.json`
2. Implement `src/orchestration.py` which loads `vocabulary.json` and calls each submodule
3. Each submodule uses its prompt from `prompts/` with vocabulary injected at the marked placeholders
4. The orchestration layer returns normalized queries + a structured transformation log
